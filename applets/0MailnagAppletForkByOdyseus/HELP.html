Descripción

Este applet es una modificación del applet Mailnag, por Hasan Yavuz (también conocido como hyOzd).

Diferencias con el applet original

  • Agregado atajo de teclado para abrir y cerrar el menú.
  • Agregada opción para mantener un solo sub-menú abierto en todo momento.
  • Eliminado código obsoleto. Principalmente características que ya no existen en JavaScript moderno.
  • Agregadas más opciones de notificación que se complementan con las opciones existentes en Mailnag.

Dependencias

  • mailnag: Mailnag es un programa demonio que comprueba servidores POP3 e IMAP por correo nuevo. Instrucciones detalladas de instalación pueden ser encontradas en el repositorio de Mailnag. Cinnamon necesita ser reiniciado luego de instalar Mailnag si el applet ya fue colocado en un panel.

Uso del applet

  • Para configurar Mailnag, ejecutar su programa de configuración desde el menú de aplicaciones, desde el menú contextual de este applet, o ejecutando el comando mailnag-config.
  • Cuentas de correo electrónico pueden ser agregadas desde el programa de configuración de Mailnag o automáticamente detectadas al instalar el complemento de Mailnag Cuentas en Línea de GNOME (GOA).
  • El complemento de Mailnag llamado Servicio DBus necesita estar habilitado para que este applet pueda usar el demonio.
  • Mailnag se puede configurar al editar su archivo de configuración en ~/.config/mailnag/mailnag.cfg.
  • Se recomienda instalar el paquete de dependencia opcional de Mailnag gir1.2-gnomekeyring-1.0 (o como se llame el paquete en una distribución dada) para que Mailnag use el anillo de claves para almacenar las contraseñas.
  • El complemento Notificaciones LibNotify de Mailnag no puede agregar acciones de notificación como marcar correos como leídos o abrir el cliente de correo directamente desde la notificación. Estas funciones solo están disponibles para Gnome (una limitación en el propio complemento). Este applet implementa sus propias notificaciones configurables para evitar dichas limitaciones. No hace falta decir que si uno elige usar las notificaciones proporcionadas por este applet, el complemento Notificaciones LibNotify debe estar deshabilitado para evitar recibir notificaciones duplicadas.

Description

This applet is a fork of Mailnag applet by Hasan Yavuz (a.k.a. hyOzd).

Differences with the original applet

  • Added keyboard shortcut to be able to open/close the menu.
  • Added option to keep just one sub-menu open.
  • Removed obsolete code. Mostly features that doesn't exist anymore in modern JavaScript.
  • Added more notification options that complements with the options existing in Mailnag's.

Dependencies

  • mailnag: Mailnag is a daemon program that checks POP3 and IMAP servers for new mail. Detailed installation instructions can be found on Mailnag's repository. Cinnamon needs to be restarted after installing Mailnag if the applet was already placed in a panel.

Usage

  • To configure Mailnag, launch its configuration program found in the applications menu, or from this applet context menu or by running the mailnag-config command.
  • E-mail accounts can be directly added through Mailnag's configuration program or automatically detected by installing Mailnag GNOME Online Accounts plugin.
  • The Mailnag plug-in called DBus Service needs to be enabled for this applet to be able to use the daemon.
  • Mailnag can be configured further by editing its configuration file at ~/.config/mailnag/mailnag.cfg.
  • It is recommended to install the Mailnag's optional dependency package gir1.2-gnomekeyring-1.0 (or whatever the package is called in a given distribution) so Mailnag uses the keyring to store passwords.
  • The Mailnag's LibNotify Notifications plug-in cannot add notification actions such as mark mails as read or open mail client directly from the notification. These features are available only for Gnome (a limitation hard-coded into the plug-in itself). This applet implements its own configurable notifications to bypass such limitations. Needless to say, if one choses to use the notifications provided by this applet, the LibNotify Notifications plug-in must be disabled to avoid receiving duplicated notifications.
