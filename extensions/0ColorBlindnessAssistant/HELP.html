Descripción

Asistente de ceguera al color es una extensión para Cinnamon que provee herramientas para asistir a usuarios con deficiencia de visión del color.

Referencias básicas

  • Acromatopía (monocromatismo de varilla): Ceguera total o casi total del color. Wiktionary
  • Acromatopía (monocromatismo de cono azul): Ceguera total o casi total del color. Wiktionary
  • Protanopía: Deficiencia visual de los primeros colores (rojos). Wiktionary
  • Deuteranopía: Deficiencia visual de los segundos colores (verdes). Wiktionary
  • Tritanopía: Deficiencia visual de los terceros colores (azules). Wiktionary

Características

Esta extensión provee tres características principales:

  • Compensación de la ceguera al color: Un usuario puede aplicar un efecto (específico a su patología) que le ayude a diferenciar mejor los colores.
  • Simulación de ceguera al color: Un desarrollador puede aplicar un efecto que le mostrará cómo una persona con deficiencia de visión del color ve ciertas combinaciones de colores.
  • Nombre de color: Esta función es útil para saber el nombre del color que está inmediatamente debajo del cursor.

Uso

  • Las funciones de esta extensión son accesibles en su totalidad a través de atajos de teclado.
  • Al menos dos atajos de teclado deben configurarse para usar todas las funciones proporcionadas por esta extensión. Uno para abrir el inspector de color y otro para abrir el daltonizador.
  • Se pueden configurar atajos de teclado adicionales para aplicar efectos específicos a la pantalla o la ventana enfocada.
  • Las miniaturas de ventanas generadas por los conmutadores Alt + Tab y las vistas previas del área de trabajo heredarán el efecto de las ventanas reales. Ver Problemas conocidos.

Descripción de herramientas

Inspector de color

El corazón de esta herramienta es una biblioteca de JavaScript llamada Nombra ese color que viene con una lista de más de 1600 nombres de colores. La siguiente información es mostrada por esta herramienta:

  • Nombre de color: Podría ser una aproximación del color de entrada o una coincidencia exacta.
  • Color de Entrada: La representación hexadecimal y RGB del color que está inmediatamente debajo del cursor del mouse.
  • Color Detectado: La representación hexadecimal y RGB del color detectado por la biblioteca Nombra ese color según el color de entrada.
  • Matiz: El nombre y la representación hexadecimal del tono de color de entrada.

Daltonizador

Daltonizador es una herramienta que permite aplicar un efecto sobre la marcha en una ventana enfocada o en la pantalla. Un efecto puede simular una deficiencia de visión del color o compensarlo. La interfaz de usuario del daltonizador se compone de las siguientes secciones:

  • Nombre de Efecto: El efecto a aplicar al actor actualmente enfocado.
  • Actor: El actor (ventana enfocada o pantalla) al cual aplicar un efecto. (*)
  • Espacio de Color: El espacio de color utilizado para realizar los cálculos de efectos. (*)

(*) Las secciones Actor y Espacio de Color podrían ocultarse desde la ventana de configuración de esta extensión por los siguientes motivos:

  1. El mecanismo de selección de actor es automático. Uno podría simplemente usar el applet Mostrar escritorio para ocultar todas las ventanas y el daltonizador apuntará a la pantalla para aplicar un efecto. Y después de aplicar un efecto, se podría usar nuevamente el applet Mostrar escritorio para mostrar todas las ventanas.
  2. Como lo entiendo (como dije, no soy un experto), el espacio de color utilizado para hacer los cálculos depende de la tecnología utilizada por el monitor/dispositivo de visualización. Si un usuario no necesita cambiar al espacio de color CIE, esta sección podría ser ocultada ya que el espacio de color sRGB es el espacio de color utilizado de forma predeterminada.

Tema de interfaz

  • Esta extensión soporta temas personalizados para estilizar sus interfaces (el inspector de color y el daltonizador).
  • El tema predeterminado (que se encuentra en CARPETA_DE_LA_EXTENSIÓN/themes/default.css) solo establece un estilo genérico para acomodar los elementos en las interfaces.
  • Para usar un tema personalizado, simplemente configurar la opción Tema como Personalizado y configurar la opción Ruta a hoja de estilo personalizada para que apunte a un archivo de hoja de estilo.
  • Para crear un tema personalizado, simplemente hacer una copia del archivo de tema predeterminado en cualquier lugar del sistema de archivos, excepto dentro de la carpeta de la extensión. El archivo de tema predeterminado tiene la lista completa de clases CSS y IDs utilizadas por esta extensión.
  • Para que los cambios del tema personalizados se reflejen mientras se modifica el archivo de tema, se puede reiniciar Cinnamon o se puede volver a cargar el tema Cinnamon.
  • Canela proporciona un comando para volver a cargar su tema. Simplemente abrir el cuadro de diálogo Ejecutar comando (Alt + F2), escribir rt y presionar Enter.

Problemas conocidos

  • Las miniaturas de ventanas en el conmutador de aplicaciones clásico no heredarán el efecto de las ventanas reales. Esta es una limitación mía; Simplemente no pude encontrar la manera de hacerlo.
  • Las miniaturas de ventanas generadas por applets/extensiones/desklets (de terceros o que vienen por defecto con Cinnamon) no heredarán el efecto de las ventanas reales. Esto sería demasiado para mí, el desarrollador de esta extensión, para cumplir (si es posible en absoluto).
  • Las ventanas cuyas decoraciones se quitaron pueden volverse invisibles cuando se intenta aplicar algún efecto. No pude encontrar una solución para esto (aparte de restaurar estas decoraciones de ventanas, eso es).

Limitaciones

  • Todos los efectos aplicados por esta extensión son transitorios. Lo que significa que, siempre que se cierre una ventana, se reinicie Cinnamon o se reinicie el sistema, todos los efectos se eliminarán/destruirán.

Sistema personalizado de configuración de xlet

Este xlet utiliza una aplicación personalizada para manejar su configuración. El sistema de configuración nativo de Cinnamon maneja aplicaciones externas para la configuración de xlets de una manera muy limitada. Para evitar estas limitaciones, establecí diferentes mecanismos según el tipo de xlet.

  • Applets: El elemento del menú contextual Configurar... está anulado para abrir la aplicación de configuración adecuada.
  • Extensiones: Al habilitar la extensión, se genera automáticamente un acceso directo (un archivo .desktop) para abrir la aplicación de configuración adecuada y se elimina cuando la extensión se deshabilita. El archivo .desktop se crea en ~/.local/share/applications, por lo que el acceso directo estará disponible en el menú de aplicaciones dentro de la categoría Preferencias.
  • Para todos los xlets: Si se abre la ventana de configuración incorrecta, estará disponible un botón para abrir la ventana de configuración correcta.

Description

Color Blindness Assistant is a Cinnamon extension that provides tools to assist users with color vision deficiency (CVD).

Basic references

  • Acromatopia (rod monochromatism): Total or almost total color blindness. Wiktionary
  • Acromatopia (blue-cone monochromatism): Total or almost total color blindness. Wiktionary
  • Protanopia: First colors (reds) vision deficiency. Wiktionary
  • Deuteranopia: Second colors (greens) vision deficiency. Wiktionary
  • Tritanopia: Third colors (blues) vision deficiency. Wiktionary

Features

This extension provides mainly three features.

  • Color blindness compensation: A user can apply an effect (specific to her/his pathology) that will (might) help her/him to better differentiate colors.
  • Color blindness simulation: A developer can apply an effect that will show her/him how a person with color vision deficiency will (might) see certain color combinations.
  • Color naming: This feature is useful for knowing the name of the color that's immediately under the cursor.

Usage

  • This extension features are accessible entirely through keyboard shortcuts.
  • At least two keyboard shortcuts need to be configured to use all features provided by this extension. One to open the color inspector and one to open the daltonizer.
  • Additional keyboard shortcuts can be configured to apply specific effects to the screen or the focused window.
  • Window thumbnails generated by the Alt + Tab switchers and workspace previews will inherit the effect of the real windows. See Known issues.

Tools description

Color inspector

The heart of this tool is a JavaScript library called Name that color that comes with a list of more than 1600 color names. The following information is displayed by this tool:

  • Color name: It could be an approximation of the input color or an exact match.
  • Input Color: The hexadecimal and RGB representation of the color that's immediately under the mouse cursor.
  • Detected Color: The hexadecimal and RGB representation of the color detected by the Name that color library based on the input color.
  • Hue: The name and hexadecimal representation of the input color hue.

Daltonizer

Daltonizer is a tool that allows to apply an effect on-the-fly to a focused window or the screen. An effect can simulate a color vision deficiency or compensate it. The daltonizer UI is composed of the following sections:

  • Effect Name: The effect to apply to the currently focused actor.
  • Actor: The actor (focused window or screen) to which to apply an effect. (*)
  • Color Space: The color space used to perform the effects calculations. (*)

(*) The Actor and Color Space sections could be hidden from this extension settings window for the following reasons:

  1. The actor selection mechanism is automatic. One could simply use the Show desktop applet to hide all windows and the daltonizer will target the screen to apply an effect. And after applying an effect, one could use again the Show desktop applet to display all windows.
  2. As I understand it (like I said, I'm no expert), the color space used to make the calculations depends on the technology used by the monitor/display device. If a user doesn't need to change to the CIE color space, this section could be hidden since the sRGB color space is the color space used by default.

GUI theme

  • This extension supports custom themes to style its GUIs (the color inspector and the daltonizer).
  • The default theme (found in EXTENSION_FOLDER/themes/default.css) only sets a generic styling to accommodate the elements in the GUIs.
  • To use a custom theme, just set the Theme option to Custom and set the Custom theme path option to point to a style sheet file.
  • To create a custom theme, just make a copy of the default theme file anywhere on the file system, except inside the extension folder. The default theme file has the complete list of CSS classes and IDs used by this extension.
  • For the custom theme changes to be reflected while the theme file is modified, either Cinnamon can be restarted or the Cinnamon theme can be reloaded.
  • Cinnamon provides a command to reload its theme. Just open the Run command dialog (Alt + F2), type rt and press Enter.

Known issues

  • Thumbnails of windows in the classic application switcher will not inherit the effect of the real windows. This is a limitation of my own; I simply couldn't find the way of doing it.
  • Thumbnails of windows generated by applets/extensions/desklets (from third-parties or that come by default with Cinnamon) will not inherit the effect of the real windows. This would be too much of an undertaking for me, the developer of this extension, to accomplish (if possible at all).
  • Windows whose decorations were removed might become invisible when trying to apply any effect. I couldn't find a fix or workaround for this (other than restoring back these windows decorations, that is).

Limitations

  • All effects applied by this extension are transient. Which means that, whenever a window is closed, Cinnamon is restarted or the system is restarted, all effects will be removed/destroyed.

Custom xlet settings system

This xlet uses a custom application to handle its settings. Cinnamon's native settings system handles external applications for xlets settings in a very limited way. To work around these limitations, I put in place different mechanisms depending on the xlet type.

  • Applets: The Configure... context menu item is overridden so it opens the proper settings application.
  • Extensions: A shortcut (a .desktop file) to open the proper settings application is automatically generated upon enabling the extension and it is removed when the extension is disabled. The .desktop file is created at ~/.local/share/applications, so it will make the shortcut available in your applications menu inside the Preferences category.
  • For all xlets types: If the wrong settings window is opened, a button to open the right settings window will be available.
