Description

Color Blindness Assistant is a Cinnamon extension that provides tools to assist users with color vision deficiency (CVD).

Basic references

  • Acromatopia (rod monochromatism): Total or almost total color blindness. Wiktionary
  • Acromatopia (blue-cone monochromatism): Total or almost total color blindness. Wiktionary
  • Protanopia: First colors (reds) vision deficiency. Wiktionary
  • Deuteranopia: Second colors (greens) vision deficiency. Wiktionary
  • Tritanopia: Third colors (blues) vision deficiency. Wiktionary

Features

This extension provides mainly three features.

  • Color blindness compensation: A user can apply an effect (specific to her/his pathology) that will (might) help her/him to better differentiate colors.
  • Color blindness simulation: A developer can apply an effect that will show her/him how a person with color vision deficiency will (might) see certain color combinations.
  • Color naming: This feature is useful for knowing the name of the color that's immediately under the cursor.

Usage

  • This extension features are accessible entirely through keyboard shortcuts.
  • At least two keyboard shortcuts need to be configured to use all features provided by this extension. One to open the color inspector and one to open the daltonizer.
  • Additional keyboard shortcuts can be configured to apply specific effects to the screen or the focused window.
  • Window thumbnails generated by the Alt + Tab switchers and workspace previews will inherit the effect of the real windows. See Known issues.

Tools description

Color inspector

The heart of this tool is a JavaScript library called Name that color that comes with a list of more than 1600 color names. The following information is displayed by this tool:

  • Color name: It could be an approximation of the input color or an exact match.
  • Input Color: The hexadecimal and RGB representation of the color that's immediately under the mouse cursor.
  • Detected Color: The hexadecimal and RGB representation of the color detected by the Name that color library based on the input color.
  • Hue: The name and hexadecimal representation of the input color hue.

Daltonizer

Daltonizer is a tool that allows to apply an effect on-the-fly to a focused window or the screen. An effect can simulate a color vision deficiency or compensate it. The daltonizer UI is composed of the following sections:

  • Effect Name: The effect to apply to the currently focused actor.
  • Actor: The actor (focused window or screen) to which to apply an effect. (*)
  • Color Space: The color space used to perform the effects calculations. (*)

(*) The Actor and Color Space sections could be hidden from this extension settings window for the following reasons:

  1. The actor selection mechanism is automatic. One could simply use the Show desktop applet to hide all windows and the daltonizer will target the screen to apply an effect. And after applying an effect, one could use again the Show desktop applet to display all windows.
  2. As I understand it (like I said, I'm no expert), the color space used to make the calculations depends on the technology used by the monitor/display device. If a user doesn't need to change to the CIE color space, this section could be hidden since the sRGB color space is the color space used by default.

GUI theme

  • This extension supports custom themes to style its GUIs (the color inspector and the daltonizer).
  • The default theme (found in EXTENSION_FOLDER/themes/default.css) only sets a generic styling to accommodate the elements in the GUIs.
  • To use a custom theme, just set the Theme option to Custom and set the Custom theme path option to point to a style sheet file.
  • To create a custom theme, just make a copy of the default theme file anywhere on the file system, except inside the extension folder. The default theme file has the complete list of CSS classes and IDs used by this extension.
  • For the custom theme changes to be reflected while the theme file is modified, either Cinnamon can be restarted or the Cinnamon theme can be reloaded.
  • Cinnamon provides a command to reload its theme. Just open the Run command dialog (Alt + F2), type rt and press Enter.

Known issues

  • Thumbnails of windows in the classic application switcher will not inherit the effect of the real windows. This is a limitation of my own; I simply couldn't find the way of doing it.
  • Thumbnails of windows generated by applets/extensions/desklets (from third-parties or that come by default with Cinnamon) will not inherit the effect of the real windows. This would be too much of an undertaking for me, the developer of this extension, to accomplish (if possible at all).
  • Windows whose decorations were removed might become invisible when trying to apply any effect. I couldn't find a fix or workaround for this (other than restoring back these windows decorations, that is).

Limitations

  • All effects applied by this extension are transient. Which means that, whenever a window is closed, Cinnamon is restarted or the system is restarted, all effects will be removed/destroyed.

Custom xlet settings system

This xlet uses a custom application to handle its settings. Cinnamon's native settings system handles external applications for xlets settings in a very limited way. To work around these limitations, I put in place different mechanisms depending on the xlet type.

  • Applets: The Configure... context menu item is overridden so it opens the proper settings application.
  • Extensions: A shortcut (a .desktop file) to open the proper settings application is automatically generated upon enabling the extension and it is removed when the extension is disabled. The .desktop file is created at ~/.local/share/applications, so it will make the shortcut available in your applications menu inside the Preferences category.
  • For all xlets types: If the wrong settings window is opened, a button to open the right settings window will be available.
